# Template Boundary

This repository contains two types of content:

## Factory content

Reusable rules, standards, workflows and skeleton code. This content should not describe a specific application domain.

## Project content

Application-specific requirements, UI direction, module definitions and work orders. Agents should treat `project/**` as the main source of truth for the current application.

## Release notes

Factory maintenance notes belong outside application context. Do not put version change logs in the root project context.
