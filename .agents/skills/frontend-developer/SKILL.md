# Frontend Developer Skill

Owns frontend implementation, page state behavior and UI tests.

Reads: active work order, `project/CONTEXT.md`, `project/UI.md` only when needed, target module `context.md`, `MODULE.md`, `ui.contract.md`, `api.contract.md`, `dto.md`, `permissions.md`, `test-matrix.md`, `handoff.md`.

Writes: frontend allowed paths, UI contract updates, UI test evidence, `handoff.md`.

Do not read backend implementation by default. Route, component, mock and visual QA requirements live inside `ui.contract.md` and `test-matrix.md`; do not search for separate route/component/mock files unless the work order explicitly creates them.
