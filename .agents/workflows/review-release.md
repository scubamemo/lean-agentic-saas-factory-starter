# Workflow

Start from `START-HERE.md`, `AGENTS.md` and `project/work-orders/active-work-order.md`. Do not scan the full repo.

## Review and release

1. Read active work order and module handoff.
2. Review changed files only.
3. Check acceptance criteria, contracts, tests, permissions, tenant isolation and error states.
4. Write PASS / PASS_WITH_WARNINGS / FAIL in `handoff.md`.
5. For release, verify `project/STATE.md` has no blocking items.
